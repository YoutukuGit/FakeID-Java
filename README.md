# 身份证号生成工具 FakeID
一个生成随机 18 位身份证号码的程序，也可检验号码是否符合 GB 11643-1999。   

仅供学习研究使用，禁止用于非法用途。
## 生成规则

生成的 18 位公民身份证号码遵循 GB 11643-1999 规定，由四部分组成：

1. 行政区划代码（6 位）
	- 来自国家行政区划代码表。
2. 出生日期（8 位）
	- 格式为 `YYYYMMDD`，例如 19900305 表示 1990 年 3 月 5 日。
3. 顺序码（3 位）
	- 顺序码为 001-999 的三位数字，奇数表示男性，偶数表示女性。
4. 校验码（1 位）
	- 计算方法：对前 17 位每位数字乘以对应的权重相加取模 11，得到的余数对应校验码表。
	- 权重（由第一位到第十七位）：{7, 9, 10, 5, 8, 4, 2, 1, 6, 3, 7, 9, 10, 5, 8, 4, 2}
	- 余数 -> 校验码 映射表：{0->'1', 1->'0', 2->'X', 3->'9', 4->'8', 5->'7', 6->'6', 7->'5', 8->'4', 9->'3', 10->'2'}
	- 举例：若前 17 位为 `11010519491231002`，按权重计算并取余得索引 2，则校验码为 `X`，完整身份证为 `11010519491231002X`。

## 检验规则

校验身份证号时程序会做以下基本检查（遵循 GB 11643-1999）：

1. 长度检查：必须为 18 位字符。
2. 格式检查：前 17 位应为数字，第 18 位可为数字或大写 `X`。
3. 校验码校验：按权重计算前 17 位的校验码，与第 18 位进行比对，完全一致则通过。

注意：程序做的是常规规则校验，并不等同于对真实身份的验证。
